## Design Context

### Users
Prospects venus évaluer si Léon peut leur livrer un site distinctif : fondateurs de startups, dirigeants de PME, agences de design, responsables marketing. Ils arrivent en journée depuis un desktop, parfois depuis un mobile. Ils ont 30 secondes pour décider s'ils prennent contact. Le portfolio EST l'échantillon — s'il est oubliable, Léon n'a pas de rendez-vous.

### Brand Personality
Confiant, architectural, méticuleux, délibéré. Ce n'est pas "le dev sympa avec des emojis" — c'est un **atelier de studio avec du goût**. Pense "maison d'édition + bureau d'architecte + laboratoire d'ingénierie". Trois mots : **monolithique, artisanal, tranchant**.

### Aesthetic Direction
Brutalisme éditorial + raffinement technique. Typographie massive qui remplit le viewport. Espaces négatifs généreux ponctués de blocs d'informations denses. Accent orange brûlé ("soleil brûlé") qui signale l'énergie sans tomber dans le neon cliché. Grille asymétrique avec rythme ressenti. **Un moment interactif signature** (grille canvas réactive au curseur + type qui se révèle au scroll).

Référence mentale : si Apple confiait ses pages produit à une fonderie typographique suisse des années 70. Anti-référence : les portfolios Vercel génériques avec Inter+noir+gradient violet.

**Theme**: Light par défaut (papier crème), dark toggle disponible. Justification : les clients non-techniques jugent en journée sur laptop, le light inspire confiance commerciale. Dark pour les visiteurs techniques du soir.

### Design Principles
1. **Type comme architecture** — La typographie n'est pas un élément, c'est le squelette. Du clamp(4rem, 12vw, 10rem) sans s'excuser.
2. **Un seul accent, utilisé avec parcimonie** — L'orange brûlé surgit 3-4 fois par écran max. Sa rareté fait sa puissance.
3. **Asymétrie contrôlée** — La grille est présente mais on la casse intentionnellement pour souligner un élément clé.
4. **Grain + chaleur** — Une texture noise subtile sur fond papier pour casser le numérique stérile.
5. **Mouvements mécaniques, pas bondissants** — Easing exponentiels, pas d'élastique cartoon. Décélérations comme un bras mécanique.
6. **Prouver, pas promettre** — Les mockups de projets doivent respirer la vraie app, pas un template.

### Typography
- **Display** : Unbounded (Google Fonts) — large géométrique, distinctive, pas dans la liste bannie
- **Body** : Hanken Grotesk (Google Fonts) — sans raffiné, personnalité sans reflex Inter
- **Mono** : JetBrains Mono (Google Fonts) — code-friendly, pas IBM Plex

### Palette (OKLCH)
Light :
- Paper : `oklch(97% 0.012 75)` — crème tiède
- Paper alt : `oklch(94% 0.014 75)`
- Ink : `oklch(17% 0.018 55)` — near-black tiède
- Ink soft : `oklch(42% 0.015 55)`
- Ink muted : `oklch(60% 0.012 55)`
- Border : `oklch(88% 0.01 70)`
- **Accent** : `oklch(67% 0.195 40)` — soleil brûlé / orange coral
- Accent soft : `oklch(67% 0.195 40 / 0.08)`

Dark :
- Ink : `oklch(11% 0.018 50)`
- Ink alt : `oklch(14% 0.016 50)`
- Text : `oklch(94% 0.013 75)`
- Border : `oklch(24% 0.015 55)`
- Accent : `oklch(72% 0.2 42)`

### Signature Moments
1. Hero avec **grille canvas interactive** qui suit le curseur + **titre massif révélé ligne par ligne**
2. Marquee de skills en continu, inversant au scroll
3. Compteurs animés à l'apparition
4. Cards de projets avec mockups vivants (déjà présents) enrichis de détails case-study
5. Custom cursor subtil qui se magnétise sur les éléments interactifs
6. Reveal choreographié au chargement (ink splash, grille qui se construit)
